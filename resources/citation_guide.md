# Citation Guide — Datasets, Tools, and Software

**Course:** EMA 6938 — Data Science for Materials  

Proper citation of datasets, databases, and software is a scientific requirement — not an optional courtesy. If you use any of these resources in your final project report, cite them. This guide gives you ready-to-use citation text for every resource used in this course.

---

## Why cite datasets and software?

- Databases and tools represent substantial research effort by their developers
- Citations allow readers to reproduce your work by accessing the same data
- Many databases (MP, JARVIS) track usage via citations to justify continued funding
- It is standard scientific practice and expected in any publication or thesis chapter

## How to cite in your final project report

### In the methods section

State which database you queried, what filters you applied, and when you accessed it:

> "Training data were obtained from the Materials Project database (Jain et al., 2013) on [date], filtered to binary oxides with GGA-PBE formation energies available, yielding 4,312 entries. Composition-based features were generated using the Magpie descriptor set implemented in Matminer (Ward et al., 2018)."

### In the reference list

Use APA 7th edition format (as shown above). Your report uses NeurIPS format — follow the NeurIPS reference style for the bibliography.

### Software versions

In an appendix or footnote, list the version of every software package used:

```
Python 3.10.0 | pymatgen 2024.x | matminer 0.9.x | 
scikit-learn 1.3.x | numpy 1.24.x | pandas 2.0.x
```

You can generate this automatically with the `watermark` package:

```python
%load_ext watermark
%watermark -v -p pymatgen,matminer,scikit-learn,numpy,pandas
```
